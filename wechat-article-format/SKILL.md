# 公众号文章排版

将 Markdown 文章按照指定主题的排版规则优化，输出带 inline style 的 HTML，可直接粘贴到微信公众号后台。

**核心原则**：不调用任何付费 API。通过理解主题规则，对 MD 源文件做结构性改写与 HTML inline 化。

---

## 工作流

```
[输入：用户的 MD 文章]
        ↓
① 确认主题（默认字节范）
        ↓
② 读取对应主题规则文件
        ↓
③ 对 MD 做通用结构优化
        ↓
④ 按主题规则应用元素样式
        ↓
⑤ 输出带 inline style 的 HTML
```

---

## Step 1：确认主题

没有指定主题时，询问文章类型并推荐：

| 文章类型                           | 推荐主题            |
| ------------------------------ | --------------- |
| 运营 / 知识分享 / 产品发布 / 功能介绍 / 科技评测 | `default`（字节范）  |
| 技术 / 开发工具介绍                    | `github-readme` |
| 深度文章 / 品牌故事 / 个人专栏             | `editorial`     |
| 深度报道 / 行业分析 / 财经评论             | `nyt-classic`   |
| 品牌故事 / 产品介绍 / 精品内容             | `apple`         |

---

## Step 2：加载主题规则

- `default` 主题 → 读取 `references/theme-default.md`
- 其他主题 → 读取 `references/theme-{主题名}.md`

---

## Step 3：通用结构优化（所有主题适用）

### 3.1 标题层级规范

| 层级        | 使用规则                       |
| --------- | -------------------------- |
| `#`       | **禁止在正文使用**，文章标题在公众号后台单独填写 |
| `##`      | 正文主标题（每篇 2–5 个）            |
| `###`     | 小节标题                       |
| `####` 以下 | 慎用，公众号读者很少需要四层结构           |

操作：将所有 `# 标题` 降级为 `## 标题`。

### 3.2 段落节奏

- 每段不超过 5 句，超长段落拆分
- 段落间保留空行

### 3.3 外链处理

微信不渲染外链：

- `[文字](url)` → `文字（见文末）`，文末追加"参考资料"列表

### 3.4 图片规范

- 本地路径图片必须替换为 HTTPS 外链
- 图片单独一行，前后各空一行
- 不支持 SVG，需转 PNG/JPG

### 3.5 不兼容语法替换

| 原语法          | 替代方案           |
| ------------ | -------------- |
| `- [ ]` 任务列表 | `- ❌` / `- ✅`  |
| `:::tip` 容器  | `> **提示：** 内容` |
| `$$` LaTeX   | 文字描述或图片        |
| 超过 5 列的表格    | 拆分或改列表         |
| 嵌套 HTML 标签   | 改为纯 Markdown   |
| `[^1]` 脚注    | 改为行内括号注释       |

### 3.6 内容结构强化

- **强调**：重点内容用 `**加粗**`
- **分节**：超过 1500 字的文章，每节加 `###` 小标题

---

## Step 4：按主题规则应用元素样式

微信公众号会剥离所有 `<style>` 标签和外部 CSS，**所有样式必须写成 inline style**。

读取主题规则文件后，对每个 Markdown 元素生成对应的 inline style HTML。

---

## Step 5：输出交付

**输出内容**：

- 带 inline style 的 HTML（代码块包裹，可直接粘贴到公众号后台"源码"模式）
- 改动说明：标题降级 N 处、段落拆分 N 处、外链转换 N 处、不兼容语法修复 N 处

**粘贴方式**：公众号后台 → 新建图文 → 点击"<>" 源码模式 → 粘贴
